## Contributing to Talketeer

Thanks for your interest in contributing to Talketeer!

Talketeer is a **feature-complete, production-grade real-time chat application**
originally designed and implemented by the project maintainer.

During **SWoC 2026**, contributions are **strictly review-gated and scope-limited**
to preserve stability, maintainability, and learning integrity.

Please read this document carefully before opening an issue or pull request.

## Before You Start

1. Check existing issues to avoid duplicates
2. Comment on the issue you want to work on
3. **Wait for explicit confirmation from the maintainer**
4. Start work **only after approval**

Unapproved pull requests increase review overhead and may be closed immediately.

## Branching & Commits

- `main` is the stable, production branch
- `dev` is the active development branch

All contributions must:
- Create a new branch from `dev`
- Open pull requests **only against** `dev`
- Keep PRs minimal and focused
- Address **one logical change per PR**

Pull requests opened against `main` will be closed without review.

## Commit Messages
There is no strict commit message format.

However:
- Commit messages must clearly describe what changed
- Vague messages like `fix stuff`, `update`, or `changes` are discouraged

Clear intent > rigid format.

## Pull Request Guidelines
Your PR must:

- Clearly explain **what** changed and **why**
- Reference the relevant issue (if applicable)
- Avoid unrelated formatting or cleanup
- Follow existing code patterns

The maintainer may request changes, ask for clarification, or close the PR if it
does not align with project goals or SWoC expectations.
